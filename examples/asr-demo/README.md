# asr-demo — v0.5 offline pipeline walkthrough

A self-contained, fully runnable example that exercises every v0.5 pipeline (ASR → text cleaning → embedding → moderation) end-to-end **without any hosted API and without downloading any model**. Reading this README + running `bash run_demo.sh` is the fastest way to see what DLRS v0.5 produces.

## What's in this folder

| File | What it is | Pipeline stage that consumes it |
| --- | --- | --- |
| `manifest.json` | DLRS record manifest (schema 0.3.0, fictional subject). | starting point — every pipeline reads `record_id` from here |
| `public_profile.json` | What the public registry would surface. | none, but validates against `schemas/public-profile.schema.json` |
| `run_demo.sh` | Driver script that walks all four pipelines. | — |
| `artifacts/raw/audio/voice_demo.wav` (gitignored) | A ~0.3 s silent 16 kHz mono WAV (~10 KB). Regenerated by `run_demo.sh` on every run. | ASR pipeline reads it; the dummy backend hashes the file deterministically. |
| `derived/` (gitignored) | Output of `run_demo.sh`. Regenerated on every run. | — |

DLRS is pointer-first, so **no raw audio is committed to git**. The placeholder WAV is regenerated deterministically (silence, fixed sample rate / depth) by `run_demo.sh` step 0; the checksum in `manifest.json` (`6e00afcc0bfb...`) is fixed against that generator output, so a clean checkout + one `bash run_demo.sh` is enough to satisfy the manifest's hash invariant. The `dummy` ASR backend never decodes audio anyway — it reads the file's SHA-256 only — so silence vs. real speech is irrelevant for the offline walkthrough. To exercise actual transcription, see "Real backends" below.

## Quick start (offline, no installs beyond `tools/requirements.txt`)

```bash
# from the repo root, after pip install -r tools/requirements.txt
bash examples/asr-demo/run_demo.sh
```

This runs:

1. **ASR.** `dummy` backend → `derived/asr/voice_demo.transcript.json` + `.descriptor.json`. Deterministic transcript derived from the audio file's SHA-256, so the demo is reproducible bit-for-bit.
2. **Text cleaning.** Reads the transcript JSON, applies NFKC normalisation + the v0.5 redaction rules, writes `derived/text/voice_demo.clean.txt`, `.redactions.json`, and a descriptor.
3. **Vectorization.** `hash` backend (deterministic 64-dim vectors derived from sha256(chunk_id ‖ text); unit-normalised so cosine distance works) → `derived/vectorization/voice_demo.index.json` + descriptor. No model download.
4. **Moderation.** Built-in v0.5 policy → `derived/moderation/voice_demo.moderation.json` + descriptor. Outcome on the demo input is `pass`.

After it finishes you should see nine files under `derived/` (four artefacts + four descriptors + one redactions sidecar).

## Real backends (optional)

Both real backends are deliberately opt-in so first-time readers aren't told to wait for a 400 MB model download.

```bash
# faster-whisper, CPU int8. Installs ~200 MB the first time.
pip install faster-whisper
REAL_ASR=1 bash examples/asr-demo/run_demo.sh

# sentence-transformers (all-MiniLM-L6-v2 by default, ~80 MB cached).
pip install sentence-transformers
REAL_EMBED=1 bash examples/asr-demo/run_demo.sh

# Both at once:
REAL_ASR=1 REAL_EMBED=1 bash examples/asr-demo/run_demo.sh
```

Note that with the silent demo WAV the real ASR backend will produce an empty transcript — which is the *correct* behaviour. Replace `artifacts/raw/audio/voice_demo.wav` with your own short clip to see a non-empty result, and update `manifest.json#/artifacts[0]/checksum` to the new SHA-256 (run `sha256sum artifacts/raw/audio/voice_demo.wav`).

## Optional: push embeddings into a local Qdrant

```bash
docker run -p 6333:6333 -v "$(pwd)/qdrant-data:/qdrant/storage" qdrant/qdrant
python tools/run_pipeline.py vectorization \
  --record examples/asr-demo \
  --backend hash \
  --qdrant-url http://127.0.0.1:6333
```

The collection is named after `record_id` (`dlrs_EXAMPLE_asr_demo`) and each point payload carries the `record_id`, source pointer, char span, text-excerpt SHA-256, backend, and model id — enough to cite a chunk back to a record + offset without re-reading the index.json.

## What this example deliberately is *not*

- **Not a benchmark.** The audio is silent and the policy lexicon is small; latency / accuracy numbers from this demo aren't representative.
- **Not a real subject.** The manifest is marked `subject.type=self` with `legal_name="PRIVATE"` and the consent fields point at `mailto:privacy@example.org`. Don't reuse the record id for anything real.
- **Not a substitute for `docs/PIPELINE_GUIDE.md`.** Once you've run the demo, that guide is where the design rationale and per-pipeline invariants live (issue #38).
