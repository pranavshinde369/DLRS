# Governance

DLRS Hub 采用三层治理：自动化准入、高风险人工复核、仲裁与复议。

## 角色

| 角色 | 权限 |
|---|---|
| Maintainer | 合并标准、模板和工具修改 |
| Reviewer | 审核档案、签发 badge、处理公开索引 |
| Trust & Safety | 冻结争议档案、处理投诉与下架 |
| Legal Advisor | 审阅未成年人、逝者、跨境、商业使用等高风险情况 |
| Participant | 上传、撤回、导出、删除自己的档案 |

## Badge

| Badge | 含义 |
|---|---|
| `verified-consent` | 已验证同意证据 |
| `public-data-only` | 仅公开资料模式，不提供高拟真克隆 |
| `restricted-runtime` | 只允许私有或受限运行 |
| `cross-border-blocked` | 不允许跨境复制或推理 |
| `memorial-review-required` | 逝者或纪念档案，需人工复核 |

## 合并规则

- 标准和 Schema 变更至少需要 2 名维护者批准。
- 新公开人类档案必须先通过自动校验，再由人工复核。
- 未成年人档案默认不得公开索引。
- 投诉触发后，维护者可以先冻结相关档案，不等待完整裁决。
